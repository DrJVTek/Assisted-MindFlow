# Implementation Plan: Inline LLM Response Display

**Branch**: `009-inline-llm-response` | **Date**: 2025-11-22 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/009-inline-llm-response/spec.md`

## Summary

This feature transforms LLM node interactions by automatically launching LLM operations when nodes are created and displaying question + markdown-formatted response inline within the node UI. Primary requirements:
- Auto-launch LLM on node creation (eliminates manual "Ask LLM" trigger for new nodes)
- Dual-pane layout: question at top, streaming markdown response below
- Vertical scrollbar for long responses
- Font size and node dimension adjustment
- Preserve "Ask LLM" for manual regeneration

**Technical Approach**: Extend existing React node components with auto-trigger hook on creation, integrate markdown renderer library, enhance node UI with split layout and scroll container, leverage existing LLM operations store and streaming infrastructure from Feature 007/008.

## Technical Context

**Language/Version**: TypeScript 5.x (frontend), Python 3.11+ (backend)

- Frontend: React 18.x, react-markdown, ReactFlow, Zustand (state)

**Target Platform**: Web (Chrome/Firefox/Edge), multiplatform desktop (Windows + Linux)

**Performance Goals**:
- Auto-launch LLM within 500ms of node creation
- Markdown rendering <100ms for responses up to 10k chars
- Smooth scrolling at 60fps
- Node resize <100ms with content reflow
**Constraints**:
- Must preserve existing LLM configuration system (Feature 007)
- Must not break existing "Ask LLM" functionality

- Multiplatform (Windows + Linux) per CLAUDE.md
**Scale/Scope**:
- 10+ concurrent LLM streams (existing Feature 008 support)
- Responses up to 50k characters
- Node canvas with 100+ nodes

## Complexity Tracking

> **Fill ONLY if Constitution Check has violations that must be justified**

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| III. Explicit Operations (auto-launch) | Auto-launch is core UX requirement from spec - users expect automatic response generation without manual trigger | Manual "Ask LLM" only: Defeats primary value proposition. User story P1 explicitly requires auto-launch to differentiate from existing manual workflow. |

**Justification**: Auto-launch "magic" is intentional design per user requirement. Transparency maintained via:
1. User explicitly creates node (intentional action)
2. Streaming response visible immediately (no hidden processing)
3. Operation tracked in existing LLM operations store (auditable)
4. "Ask LLM" available for manual control (escape hatch)

This is acceptable pragmatic violation - user intent is clear (create node with question) and system response is predictable (generate answer).
